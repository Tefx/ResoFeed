# UI Runtime Fresh Review Follow-up Repair Contract Matrix

Date: 2026-05-18

Scope: acceptance-only repair contract for the current-operation / reprocess-library visibility problem and fresh review findings FR-01 through FR-08 from `docs/audits/ui-preview-runtime-fresh-review-2026-05-18.md`. This matrix defines required acceptance, proof obligations, affected surfaces, and downstream ownership only. It does not ship product behavior, test assertions, runtime polling, handlers, CSS, storage, or implementation logic.

Authoritative outcome: current operation status is visible only where contextual operation work is in scope, especially during long-running library reprocess in Source Ledger and while the `RESOFEED` utility menu is open. Conflict feedback uses the canonical current operation rather than stale/opportunistic text. Idle operation state clears instead of creating persistent top-chrome idle chrome.

## Authority and Boundary Lock

Remediation MUST preserve these constraints:

- UI authority: `docs/DESIGN.md` defines the low-chrome app shell, `RESOFEED` utility menu, Source Ledger, Owner Token Prompt, canonical Source Ledger actions, and forbidden UX concepts. In particular, Source Ledger is the flat source-management surface, not a settings dashboard (`docs/DESIGN.md:569-600`), and the app shell exposes `TODAY` / `SOURCE LEDGER` through the discreet `RESOFEED` menu rather than persistent top-level links (`docs/DESIGN.md:370-375`, `431-435`).
- Preview authority: `docs/ui-preview.html` is the static DOM/copy preview for Source Ledger and utility surfaces; FR-07 and FR-08 below record required preview repair where the preview itself currently drifts from `docs/DESIGN.md`.
- Architecture authority: `docs/ARCHITECTURE.md` requires one Go process, one SQLite+FTS store, thin HTTP/MCP transports, no extra services, no job/queue/history/activity-ledger surfaces, and process-local current operation only (`docs/ARCHITECTURE.md:13-21`, `51-75`, `200-211`, `949-1005`, `1587-1611`).
- Repository boundary: `AGENTS.md:11-36` forbids vector/RAG, accounts/OAuth/per-agent registries, service/repository/DI layers, settings dashboards, folders/tags/unread/mark-all-read, and source-category management.
- Authentication boundary: all `/api/*` routes require owner-token authorization (`docs/ARCHITECTURE.md:818-823`), `actor_id` is provenance/idempotency only (`docs/ARCHITECTURE.md:1437-1444`), and operation triggers such as `POST /api/runtime/reprocess-library`, `POST /api/ingest`, and `POST /api/sources/{id}/fetch` remain owner-token gated (`docs/ARCHITECTURE.md:1231-1251`, `1374-1420`).

## Forbidden Concepts Acceptance

Acceptance explicitly forbids these remediation shortcuts:

- No persistent top-chrome idle current-operation status; idle clears to no contextual status display.
- No job dashboard, progress dashboard, retry dashboard, queue, queue history, command history, activity ledger, operation ledger, ingest ledger, task table, durable job rows, portable operation receipts, sync/merge controls, sidecar worker, extra service, event bus, service/repository/DI layer, settings dashboard, preference center, onboarding wizard, source categories, folders, tags, unread counts, mark-all-read, archive bin, drag ordering, ranking sliders, second Source Ledger URL paste field, or extra admin/worker process.
- No owner-token weakening: all operation endpoints and operation-triggering controls continue to require owner-token-backed `/api/*` or `/mcp` authorization; `actor_kind` / `actor_id` remain provenance and idempotency metadata only.
