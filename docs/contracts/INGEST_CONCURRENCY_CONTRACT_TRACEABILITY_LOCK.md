# Ingest Concurrency Contract Traceability Lock

Status: contract-only traceability artifact. This file intentionally ships no product implementation logic, runtime route behavior, schema migration, durable work machinery, MCP ingest/fetch tools, or UI rendering behavior.
Authoring step: `ica-contract-traceability-lock`.
Contract purity: enforced.

## Scope and Non-Intersection Rules

This register owns traceability for the accepted source-scoped ingest concurrency acceleration contract only. It maps requirements to expected-red tests or later verification gates; it does not implement those requirements.

Global constraints preserved across every row:

- ResoFeed remains one `cmd/resofeed` binary with flat `internal/resofeed` product logic, SQLite/FTS5 storage, and direct in-process coordination.
- Source display/title remains canonical `sources.title`; this contract must not introduce `feed_title` in storage, HTTP, MCP, UI, fixture fields, aliases, or client fallbacks.
- Manual ingest and manual source fetch HTTP request bodies remain exactly `{}` with no idempotency key, query parameters, force flags, or queued/pending command receipt.
- Manual row fetch, manual all-source ingest, and background ingest share source-scoped in-memory leases; same-source duplicates conflict fast, unrelated sources may overlap within bounded capacity, and all-source runs drain selected idle sources through bounded in-request workers.
- Source capacity exhaustion is external-pressure-only for all-source manual/background runs: it is not emitted for idle sources merely waiting behind that same run's own worker limit.
- Current operation remains ephemeral and contextual; state import/restore are short unrepresented global-exclusive mutations and must not add `state_import` or `state_restore` current-operation kinds.
- No durable queues, jobs, workers, sidecars, event buses, operation dashboards, command/activity/reading history, retry panels, settings drift, or persistent pending work are permitted.
- MCP mirrors existing language/source/runtime-operation semantics only. This contract does not authorize MCP ingest or fetch trigger tools.
