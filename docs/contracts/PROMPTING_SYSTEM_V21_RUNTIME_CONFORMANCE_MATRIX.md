## Scope and Non-Intersection Rules

This artifact locks requirement-to-checklist traceability only. It does not authorize implementation scope expansion.

Global constraints preserved across every row:

- one deployable Go binary (`cmd/resofeed`) remains the only runtime;
- SQLite plus FTS5 remains the only durable storage/retrieval substrate;
- OpenRouter remains the sole LLM backend and the LLM remains a bounded JSON-in/JSON-out transformer;
- Go owns prompt compilation, validation, runtime/provider classification, retries, persistence decisions, and public status mapping;
- no durable prompt/model preference state, prompt template library, model history, job queue, activity ledger, provider registry, sidecar worker, vector store, embeddings, RAG, translation comparison panel, or automatic stored-content rewrite on language switch is authorized;
- deterministic compiler/normalizer/validator logic belongs in core functions with explicit inputs; OpenRouter, HTTP, SQLite, browser, localStorage, logs, `/doctor`, and runtime metadata effects are shell-only;
- request-scoped prompt/model values must never be written to SQLite runtime metadata, state export/import, source/item durable metadata, localStorage, provider config, logs, `/doctor`, or user-visible history.

## Status Legend

- `PROVEN`: requirement is directly established by cited documents and this row maps it to concrete proof obligations; not a claim that runtime implementation has passed unless evidence says so.
- `UNPROVEN`: accepted/pending contract target requiring downstream implementation or verification evidence.
- `BLOCKED`: cannot be verified without missing authority or runtime capability.
- `EXCLUDED_OR_DEFERRED`: deliberately not implemented/currently pending per docs; row prevents false implementation claims.
- `LEGACY_SUPERSEDED`: retained for historical v2.1 traceability only; active v2.2/Tavily implementation must use the current authoritative docs instead.
