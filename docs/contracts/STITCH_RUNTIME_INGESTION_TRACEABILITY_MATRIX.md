# Stitch Runtime Ingestion Traceability Matrix

Status: contract traceability artifact; not runtime proof.
Authoring step: `stitch-runtime-contract-traceability-matrix`
Authority posture: review-approved Stitch deltas are implementation evidence inputs only. They do not override `CONSTITUTION.md`, `docs/PRD.md`, `docs/ARCHITECTURE.md`, `docs/DESIGN.md`, `docs/DESIGN_VISION.md`, or active contracts.

## Scope and Non-Intersection Rules

This matrix maps the accepted Stitch design-ingestion deltas to downstream runtime implementation, expected-red test, audit, or explicit exclusion obligations. It does **not** authorize product code changes, new product concepts, CSS/runtime changes, or any UI behavior beyond the cited ResoFeed boundaries.

Global constraints preserved across every row:

- Stitch artifacts are design evidence/checkpoint inputs only; conflicts are resolved in favor of Constitution/PRD/Architecture/DESIGN/DESIGN_VISION/contracts.
- `docs/ui-preview.html` is static design evidence only and must not be counted as runtime liveness proof.
- Runtime proof requires live DOM/a11y/network/state evidence, an automated runtime test, or a downstream audit receipt.
- Product primitives remain Inspect, Resonate, and Steer; search is lexical desk review, not chat/RAG/answer mode.
- Feed remains a compact scan surface; Inspector owns structured generated reading fields and provenance detail.
- Source Ledger is a flat operational roster with bracket actions and current-operation text, not settings/admin/job/history/sync UI.

## Downstream Step Reference Registry

The following downstream step IDs are traceability targets only; this artifact does not mutate plan state.

- `stitch-runtime-authority-boundary-audit` — verifies authority hierarchy, Stitch/static-preview non-authority, and rejected-drift exclusions.
- `stitch-runtime-visual-delta-implementation` — applies and verifies warm archival palette, mono operational chrome, flat density, split workspace, and responsive/bilingual visual coverage.
- `stitch-runtime-surface-inventory-dom` — verifies Owner Token, First-Use Empty, RESOFEED menu, TODAY, SOURCE LEDGER, Current Operation, Search Retrieval, Inspector, and full mobile Inspector runtime surfaces.
- `stitch-runtime-product-primitives-search` — verifies Inspect/Resonate/Steer only and lexical search filtered-slice behavior.
- `stitch-runtime-content-boundary` — verifies Feed/Inspector generated-content separation, Chinese section labels, provenance/source evidence, and item re-ingest placement.
- `stitch-runtime-source-ledger-contract` — verifies flat ledger roster, manual ingest/fetch bracket actions, current-operation text, state import/export, and no admin/settings/job drift.
- `stitch-runtime-localization-provenance` — verifies generated Chinese labels and literal preservation of source identifiers, URLs, titles, and model IDs.
- `stitch-runtime-steer-state-a11y` — verifies Steer operational states, accessibility live regions, and no chat/rule-builder drift.
- `stitch-runtime-doctor-raw-text` — verifies `/doctor` remains raw text and not a dashboard.
- `stitch-runtime-motion-negative-audit` — verifies no CSS animation/transition drift on bracket actions, source-ledger status, or manual ingest controls.

## Status Legend

- `OWNED`: accepted runtime obligation mapped to one concrete downstream step, checklist item, and required evidence field.
- `EXCLUDED`: rejected drift or non-runtime-proof usage explicitly excluded by authority citation.
- `DEFERRED`: intentionally postponed obligation with non-intersection proof that current runtime behavior is not authorized or claimed.
- `AMBIGUOUS`: authority conflict or missing owner requiring escalation before implementation.
