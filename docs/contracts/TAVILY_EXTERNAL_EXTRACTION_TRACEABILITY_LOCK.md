# Tavily External Extraction Contract Traceability Lock

Status: contract-only traceability artifact. This file intentionally ships no runtime Tavily implementation, no Tavily HTTP client, no schema migration, no UI rendering behavior, no product-scope expansion, and no vectl lifecycle mutation.
Authoring step: `contract-traceability-lock`.
Contract purity: this artifact consolidates the existing authoritative references for downstream expected-red tests, implementation, and final verification. It does not authorize architecture decisions beyond those references.

## Scope and Non-Intersection Rules

This register owns traceability for optional Tavily-backed source-text recovery only. Every requirement below must be interpreted through the existing ResoFeed boundaries:

- ResoFeed remains one `cmd/resofeed` process serving static UI, JSON HTTP, MCP Streamable HTTP, SQLite migrations, and background ingest.
- Product behavior remains flat `internal/resofeed` logic with SQLite/FTS5 as the only storage/retrieval substrate.
- Tavily is source-acquisition infrastructure only: it may recover source evidence, but OpenRouter remains the sole LLM-backed JSON transformer for generated item understanding.
- Tavily is attempted only after local readable extraction and RSS/source-backed fallback fail for an eligible selected article URL; it is not an alternate-source discovery, summarization, ranking, translation, or classification provider.
- HTTP and MCP expose the same product operations and canonical schemas. There is no Tavily-only agent concept unavailable to the human HTTP/UI path.
- Source evidence is current item state, not portable state and not history. Portable state remains active sources, active steering rules, and currently resonated items.
- UI may render provenance/status labels and source-backed Text evidence, but must not add provider settings, dashboards, wizards, or provider configuration surfaces.
- Forbidden drift remains forbidden: no provider registry, service layer, durable queue, worker, sidecar, browser-agent scraping, login-cookie scraping, settings dashboard, RAG/vector dependency, durable extraction history, or activity ledger.
