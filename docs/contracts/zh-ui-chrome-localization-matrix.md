# zh UI Chrome Localization Requirement Matrix

Status: proposed proof artifact for `zh-ui-contract-matrix`  
Scope: frontend UI chrome localization requirements and design exception decisions for the post-closure Chinese UI repair.  
Authority: `docs/DESIGN.md`, `docs/ARCHITECTURE.md`, `docs/POST_CLOSURE_REINGEST_MODEL_I18N_REPAIR_CONTRACT.md`, and the current frontend chrome surfaces listed below.

## Purpose

This artifact records the broad Chinese UI chrome localization requirement-to-checklist matrix before implementation. It exists so downstream implementers and verifiers can prove per-surface coverage instead of relying on generic claims such as "Chinese support verified." It is intentionally a contract/proof matrix, not an implementation design for a framework-level i18n system.

## Binding Constraints

- No new framework-level i18n system, frontend dependency, backend service, storage migration, sidecar, worker, queue, or extra runtime process is authorized by this matrix.
- Preserve the single Go binary architecture: the existing Svelte frontend is served by `cmd/resofeed`, and all localization repair covered here is frontend chrome/test/documentation work unless an already-authorized API contract requires tiny test/doc alignment.
- SQLite+FTS5 remains the only storage/retrieval substrate. This matrix does not authorize embeddings, vector databases, RAG, semantic answer surfaces, state mergers, or sync/ledger additions.
- The following values remain literal provenance or operational anchors and must not be translated, summarized, transliterated, beautified, or rewritten: source identifiers, source titles, source URLs, item URLs, canonical URLs, original links, model IDs, actor/provenance identifiers, timestamps, technical IDs, `/doctor`, `RESOFEED`, `TODAY`, `SOURCE LEDGER`, and `INSPECTOR` unless an individual row explicitly scopes only adjacent explanatory/status chrome as translatable.
- Source/provenance anchors must retain `translate="no"` or an equivalent non-translation mechanism wherever the current surface already marks them or the repair touches them.

## Design Exception Decision: Operational Tokens vs. Localized Chrome

Decision: retain `RESOFEED`, `TODAY`, `SOURCE LEDGER`, `INSPECTOR`, and `/doctor` as brand/system chrome and localize adjacent explanatory, accessible-name, status, button-state, placeholder, empty-state, and error text when processing language is `zh`.

Rationale: `docs/DESIGN.md` defines the app as low-chrome archival-index software and explicitly says product chrome should use operational labels such as `RESOFEED`, `TODAY`, `YESTERDAY`, `SOURCE LEDGER`, `INSPECTOR`, `/doctor`, and raw status strings. `docs/ARCHITECTURE.md` separately makes source identifiers preservation anchors because users and agents need exact provenance even when readable content is processed in another language. Therefore, localization must make the user-facing surrounding copy understandable in zh without weakening the invariant that system labels and provenance anchors remain stable.

Trade-off accepted: zh mode may contain a small number of English uppercase operational tokens by design. This sacrifices full linguistic uniformity to preserve stable navigation/provenance anchors and avoid creating a new product vocabulary that conflicts with the design system. This decision fails if a later product/design authority explicitly reclassifies one of those operational tokens as ordinary translatable prose.

## Source Surfaces Read

- `web/src/routes/+page.svelte`: shell, Steer, `RESOFEED` surface menu, language/reprocess controls, route preview, back commands, status lines, Feed/Search/Inspector/Source Ledger wiring.
- `web/src/routes/components/Feed.svelte`: feed region labels, item open/resonate aria, metadata labels, load-more action, shared item-anatomy helper consumption, and `translate="no"` source title behavior.
- `web/src/routes/components/SearchRetrieval.svelte`: search heading, filters, status/result counts, result metadata, empty state, result inspect/resonate aria, and source title non-translation.
- `web/src/routes/components/SourceLedger.svelte`: Source Ledger actions/status/empty state, source row grammar, OPML import, fetch/delete/details buttons, and `translate="no"` source title/source URL behavior.
- `web/src/routes/components/StatePortability.svelte`: export/import bracket actions, file input accessible names, warning, and status messages.
- `web/src/routes/components/OwnerTokenPrompt.svelte`: owner token prompt copy, submit action, and rejected-token status.
- `web/src/routes/components/FirstUseEmptyState.svelte`: first-use explanatory lines inside the normal shell.
- `web/src/routes/components/item-anatomy.ts`: shared item labels for time groups, ages, extraction labels, provenance labels, priority/quality labels, and summary-unavailable fallback.
- `web/src/routes/components/Inspector.svelte`: Inspector chrome, provenance/status lines, source disclosure, grouped-source labels, re-ingest panel, model-list diagnostics, residual English copy, source URL/title non-translation, and successful existing zh helpers.

## Non-Translation Proof Register

| preserved class | examples | must remain literal because | required downstream proof |
| --- | --- | --- | --- |
| Product/system operational tokens | `RESOFEED`, `TODAY`, `SOURCE LEDGER`, `INSPECTOR`, `/doctor` | DESIGN.md assigns these as product chrome/operational labels, not ordinary prose. | DOM text table showing tokens unchanged in zh mode while adjacent labels/status text localize. |
| Source identity values | source title, source URL, feed URL, canonical URL, original link, item URL | ARCHITECTURE.md decision 12 and DESIGN.md Source Identifiers define them as trust/provenance anchors. | DOM/attribute table showing literal values and `translate="no"` or equivalent where rendered. |
| Model/provider identifiers | OpenRouter model IDs, `account_default`, provider/runtime diagnostic identifiers | Architecture treats OpenRouter IDs as runtime/provider identifiers, not user prose. | Re-ingest/model-selector proof that labels may localize but option values/IDs stay unchanged. |
| Actor/provenance/technical identifiers | `actor_id`, `agent:<id>`, `story_key`, `duplicate_of_item_id`, timestamps, source/item IDs | These are provenance/idempotency/technical anchors, not localized copy. | Inspector/feed/search/ledger proof that human-readable labels localize while identifier payload remains exact. |
| Raw diagnostics/status codes | `err:`, internal route `/doctor`, backend error code fragments where shown raw | DESIGN.md allows raw operational strings and architecture requires terse diagnostics. | Error/status proof distinguishes translated explanatory copy from raw codes/prefixes retained by design. |

## Downstream Verification Minimum

Later gates should reject evidence unless it includes all of the following:

1. Per-surface runtime DOM/screenshot proof for shell, feed, search, source ledger, state portability, owner token prompt, first-use empty state, Inspector residuals, and shared item-anatomy labels.
2. A `nontranslated_identifiers_table` showing exact literal source titles/URLs/model IDs/actor or provenance identifiers before and after zh mode.
3. Accessible-name proof for key controls and aria-live/status regions, not just visible text.
4. Expected-red or equivalent test output demonstrating that broad chrome tests fail before implementation and are not Inspector-only.
5. Evidence that no framework-level i18n dependency, backend process, storage migration, sidecar, queue, or new runtime state was introduced.

## Open Questions

- Non-blocking: if design later wants `TODAY`/`YESTERDAY`/`EARLIER` localized while retaining `TODAY` as a menu token, that must be decided explicitly by product/design authority. This matrix preserves current operational time-group tokens by default.
- Non-blocking: `err:` raw prefix is preserved by this matrix; explanatory error text may localize when it is not a backend raw diagnostic.
